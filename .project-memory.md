## Core Principles (from AGENTS.md)
- **Idempotency**: Re-running bootstrap must not duplicate config or break shell
- **Safety**: Never overwrite user dotfiles blindly; backup before modifying
- **Determinism**: Pin versions; make install order explicit
- **Maintainability**: Keep scripts small and named clearly; separate concerns
- **WSL2 compatibility**: Assume Ubuntu/Debian; respect WSL paths/interop

## Important Implementation Notes

### Inquirer v9 API
In v9 wurde die API geändert. Der korrekte Import ist:
```javascript
const inquirer = require('inquirer').default;
```

### Bash Script Standards
- Non-triviale Scripts: `set -euo pipefail`
- Variablen konsistent quoten
- `[[ ... ]]` statt legacy `[`
- Tool-Detection: `command -v tool >/dev/null 2>&1`

### Configuration Management
- Managed sections in dotfiles:
```text
# >>> wsl-dev-bootstrap >>>
# managed block
# <<< wsl-dev-bootstrap <<<
```

### Logging Categories
- INFO, WARN, OK, ERROR
- Keep messages short and actionable

## Installation Workflow
1. `git clone https://github.com/sjess/st4rd0tf1les_l1gh7.git ~/.dotfiles`
2. `cd ~/.dotfiles && ./start`
3. `./setup_sudo_user.sh` (als root)
4. `npm install` + `node index.js` (interaktives Setup)

## Shell Integration
- Repo wird als `.dotfiles` im Home-Verzeichnis installiert
- `.zshrc` wird auf `~/.dotfiles/shell/.wsl2` verlinkt
- Aliase/Funktionen werden in `.zshrc` geladen

## opencode Integration
- Wird in `install.sh` installiert: `curl -fsSL https://opencode.ai/install | bash`
- Konfiguration: `~/.dotfiles/.opencode/` → `~/.opencode/`
- Enthält: `opencode.json` + `skills/` Ordner

## Avoid (from AGENTS.md)
- Overwrite `.zshrc`, `.bashrc` without backup/merge strategy
- Assume root unless explicitly running under sudo
- Assume GUI availability inside WSL2
- Hardcode usernames, hostnames, Windows mount paths
- Mix unrelated responsibilities into one giant script
- Rely on fragile `sed` one-liners
